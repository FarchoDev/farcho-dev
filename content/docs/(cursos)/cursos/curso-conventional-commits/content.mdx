## Módulo 1: Fundamentos de Conventional Commits
* **¿Por qué Conventional Commits?** Explicaremos en detalle los beneficios de adoptar esta convención en tus proyectos, tanto a nivel personal como profesional.
* **Estructura básica:** Aprenderás la estructura de un mensaje de commit convencional, incluyendo el header, body y footer.
* **Tipos de commit:** Estudiaremos los tipos de commit más comunes (feat, fix, docs, etc.) y sus usos adecuados.
* **Ejemplos prácticos:** Analizaremos ejemplos concretos de mensajes de commit bien estructurados y cómo aplicarlos a tus proyectos.

## Módulo 2: Conventional Commits en la Práctica
* **Configuración de tu editor:** Te enseñaré a configurar tu editor de código (Visual Studio) para que te sugiera la estructura de los mensajes de commit y facilite la escritura.
* **Integración con GitHub:** Veremos cómo integrar Conventional Commits con GitHub para generar automáticamente changelogs y otras funcionalidades.
* **Conventional Commits y semver:** Exploraremos la relación entre Conventional Commits y Semantic Versioning (semver) para gestionar las versiones de tus proyectos de manera coherente.
* **Mejores prácticas:** Te daré consejos y recomendaciones para escribir mensajes de commit claros, concisos y útiles.

## Módulo 3: Casos de Uso Avanzados
* **Conventional Commits en equipos:** Aprenderás cómo aplicar Conventional Commits en un entorno colaborativo y cómo resolver posibles conflictos.
* **Automatización con scripts:** Descubrirás cómo automatizar tareas relacionadas con Conventional Commits utilizando scripts (por ejemplo, con Python o Bash).
* **Herramientas adicionales:** Te presentaré algunas herramientas y plugins que pueden ayudarte a trabajar de manera más eficiente con Conventional Commits.

## Módulo 4: Ejercicios Prácticos
* **Proyectos de ejemplo:** Realizaremos ejercicios prácticos en proyectos de PHP y Python (Laravel y Flask) para consolidar tus conocimientos.
* **Revisión de código:** Te proporcionaré ejemplos de código con diferentes estilos de commit y analizaremos juntos cómo mejorarlos.
* **Creación de un changelog:** Aprenderás a generar un changelog detallado y bien estructurado a partir de tus commits.

**Recursos Adicionales:**
